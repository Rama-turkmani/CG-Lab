# CG Project 3 - Interactive OpenGL Shapes

## وصف المشروع
مشروع رسوميات حاسوبية تفاعلي باستخدام OpenGL/GLFW/GLEW يتضمن:

### المتطلبات المُنفَّذة

#### 1. التفاعل - 4 أزرار للتحكم
| المفتاح | الوظيفة | التفاصيل |
|---------|---------|----------|
| **Q** | تشغيل/إيقاف الحركة | تحريك المثلثين بمسارات دائرية مختلفة باستخدام sin/cos |
| **W** | تغيير الألوان | التبديل بين 4 أنماط: الأصلي، أخضر، بنفسجي، أصفر |
| **E** | تغيير الشفافية | التبديل بين 4 مستويات: 100%، 75%، 50%، 25% |
| **R** | وضع الشبكة السلكية | التبديل بين وضع التعبئة ووضع الخطوط (Wireframe) |
| **ESC** | إغلاق البرنامج | - |

#### 2. الأبعاد - محور Z
- **المثلث الأول (أحمر):** Z = 0.0 (قريب من الكاميرا)
- **المثلث الثاني (أزرق):** Z = 0.5 (بعيد عن الكاميرا)
- تم تفعيل `GL_DEPTH_TEST` لاختبار العمق

#### 3. الشفافية (Transparency)
- تم تفعيل `GL_BLEND` مع معادلة `GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA`
- المثلث الأحمر القريب يُرسم بشفافية أعلى (alpha × 0.7) لإظهار التداخل
- يمكن التحكم بمستوى الشفافية عبر المفتاح **E**

## التقنيات المستخدمة
- **OpenGL 3.3 Core Profile**
- **GLFW** - إدارة النافذة والمدخلات
- **GLEW** - تحميل دوال OpenGL
- **GLSL Shaders** - Vertex & Fragment shaders مع uniforms

## البنية التقنية
```
Vertex Shader:
  - يستقبل الموقع (aPos) واللون (aColor)
  - يُطبّق إزاحة الحركة (uniform uOffset)

Fragment Shader:
  - يُطبّق معدِّل اللون (uniform uColorMod)
  - يتحكم بالشفافية (uniform uAlpha)
  - يستخدم clamp لضمان بقاء الألوان في النطاق الصحيح
```

## كيفية التشغيل
1. افتح المشروع في Visual Studio
2. تأكد من ضبط مسارات المكتبات (GLFW, GLEW) في إعدادات المشروع
3. اختر **x64 Debug** ثم اضغط **F5** للتشغيل
